# 🗺️ Geospatial Explorer | Next.js Client

This repository is the interactive frontend client for the **Fog Finder** and **Peak Finder** geospatial modeling tools. Built with Next.js and TypeScript, it provides a fast, responsive map interface to visualize complex elevation and weather data generated by the serverless backend.

---

## 🏗 System Architecture

This project is built with a strict separation of concerns to ensure a highly performant and scalable architecture:

1.  **Heavy Lifting (Backend):** All intensive GIS processing, elevation mathematics, and predictive fog modeling are handled by the Python backend deployed via **Modal**. This keeps the compute-heavy tasks server-side and dynamically scalable.
2.  **Lightweight Presentation (Frontend):** This Next.js client acts purely as a presentation layer. It fetches the processed data points and map layers from the Modal API endpoints and renders them smoothly on the client side using the `Map.tsx` component.

By offloading the spatial data crunching to Modal, this frontend remains incredibly lightweight, ensuring fast load times and a seamless user experience.

## 🛠 Tech Stack

* **Framework:** Next.js (App Router)
* **Language:** TypeScript
* **Styling:** Tailwind CSS / PostCSS
* **Mapping UI:** React component architecture (specifically the `Map.tsx` integration)
* **Linting & Config:** ESLint, Next config natively typed.

## 📂 Project Structure

Based on the Next.js App Router paradigm, the codebase is structured for clarity and modularity:

| Directory/File | Purpose |
| :--- | :--- |
| **`src/app/`** | The core Next.js App Router directory containing the main layout (`layout.tsx`), global styles (`globals.css`), and the root page (`page.tsx`). |
| **`src/components/`** | Reusable UI modules, notably the `Map.tsx` component responsible for rendering the geospatial layers fetched from the Modal API. |
| **`public/`** | Static assets like the `favicon.ico`. |
| **Configuration Files** | Standard modern web tooling including `tsconfig.json`, `tailwind.config` (via postcss), and `eslint.config.mjs` for strict type safety and code quality. |

## 💻 Local Development

To run the frontend client locally, you will need to link it to your Modal-deployed API.

### 1. Installation
Clone the repository and install dependencies:

```bash
git clone [https://github.com/yourusername/geospatial-frontend.git](https://github.com/yourusername/geospatial-frontend.git)
cd geospatial-frontend
npm install
```

### 2. Environment Variables
Create a .env.local file in the root directory. You will need to define your backend API URL and any required map provider tokens (if using a service like Mapbox).
```bash
NEXT_PUBLIC_BACKEND_URL="your_modal_backend_url"
NEXT_PUBLIC_MAP_TOKEN="your_map_provider_token"
```

### 3. Start the Application
Boot up the development server:
```bash
npm run dev
```
Open **http://localhost:3000** in your browser to explore the map interface

## 🔗 Backend Architecture
This frontend is decoupled and relies entirely on the Python backend for heavy GIS processing(elevation math, viewsheds, weather modeling).
